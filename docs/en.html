AI Feature Design Notes
• AI Default: Close，must be enabled in iOS Companion App Settings
• Data sent: anonymous stats only (HR delta, HRV delta, mode, duration) — zero PII
• Fallback Messages stored in App Bundle, network-independent, never crashes
• All AI outputs always end with "⚙ AI-generated content, not medical advice"
Flow D — No HealthKit Fallback
After denying or revoking HealthKit permission, the app silently degrades. The three core screens (Home / Breathing / Result) share the same components, conditionally rendered by auth state — no separate error screens. No restart needed after permission changes.

Fallback Mode Design Notes
• After denying, app silently degrades — no errors or warnings shown
• Tip (💡) shown once, dismissible, non-recurring
• The same SwiftUI View, using @Published var isAuthorized: Bool for conditional rendering
• Once enabled on iPhone, the app uses HKHealthStore.getRequestStatusForAuthorization auto-detected, no restart needed
